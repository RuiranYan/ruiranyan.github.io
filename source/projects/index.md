## 研究与模型

### OmniGen / OmniGen2 — 通用图像生成与主体驱动数据

在北京智源研究院参与 **OmniGen** 系列项目，目标是构建统一的图像生成模型，能够同时支持文本到图像生成、图像编辑以及多种视觉任务，并通过统一框架实现跨任务的知识迁移。

- 负责从零搭建主体驱动多模态数据工作流，基于 GroundingDINO + SAM / SAM2 + Outpaint 等组件，构建大规模主体驱动数据集（X2I / X2I2）。
- 在 OmniGen V1 阶段，面向静态图像场景构建主体驱动样本，使模型具备更强的主体一致性与编辑能力。
- 在 OmniGen V2 阶段，引入视频数据并通过 SAM2 对同一主体进行跨帧追踪，获得同一主体在不同时间的多种姿态，大幅缓解简单 copypaste 带来的生硬感，让模型在图像编辑、上下文生成中更自然地保持主体身份一致与细节丰富。

相关代码与项目：  
- OmniGen: https://github.com/VectorSpaceLab/OmniGen  
- OmniGen2: https://github.com/VectorSpaceLab/OmniGen2

### O1Embedder — 让检索模型先“想一想”

随着推理型大模型的发展，复杂多步推理任务的能力越来越强，但传统表征模型难以直接利用这类能力。基于这一问题提出 **O1Embedder**：

- 通过构造对下游任务有帮助的长思维链数据，将大模型的“思考过程”转化为可监督的信号。
- 使用多任务学习将推理能力与表征能力统一到同一模型中，让检索模型也能在复杂推理、zero-shot 等场景下具备更好的理解与泛化能力。

项目链接：  
- Code: https://github.com/RuiranYan/o1embedder

### 多任务推荐与工业级模型迭代

在 OPPO 与后续工作中，围绕多任务推荐模型与工业级系统迭代做了较多实践：

- 在信息流及应用商店分发搜索主场景下，对公司多任务模型 PLE 进行结构优化，引入公共专家间表征解耦损失与私有专家辅助蒸馏损失，并加入自动信息筛选模块，减轻任务间负迁移。
- 优化底层 Gate Layer 设计，通过低秩分解显著降低模型复杂度，使得模型可以支持更深的 Gate Net 与更大的 sparse embedding 规模。
- 在真实工业数据集上带来可观的 AUC 提升，并最终在线上取得显著指标收益。

## 工程与实习实践

### 字节跳动 · 抖音搜索 — 个性化预训练大模型 / LLM4Rec

在字节跳动抖音搜索多模态预训练方向，主要围绕“个性化预训练大模型”展开工作：

1. **多模态表征构建**：
   - 通过继续预训练、监督微调、后训练优化三个阶段，构建抖音域内多模态视频表征模型，相比原有纯文本表征模型，离线 Recall@10 明显提升。
   - 将多模态表征融入现有精排模型，在单列严点、双列主动点等核心指标上均取得稳定增益。

2. **量化分词与表征应用**：
   - 引入 RQ-VAE / RQKmeans 等方法，将视频表征量化为离散 token，兼顾表达能力与检索效率。
   - 设计基于 PS 回写的表征应用方案，通过 q2d 共现回写、q2d2q 两跳回写等方式引入协同信息记忆增强，在长序列与长尾 query 场景中进一步提升效果。

整体工作围绕“多模态表征 + 量化分词 + 下游应用”闭环，面向 LLM4Rec 在大型内容平台中的落地探索。

### OPPO — 信息流与应用商店多任务推荐

在 OPPO（深圳）实习期间，主要负责信息流及应用商店分发搜索主场景下多任务推荐模型的迭代：

- 在公司现有 PLE 多任务框架基础上，引入表征解耦与辅助蒸馏机制，缓解任务间负迁移问题，提升对多任务信号的利用效率。
- 优化 Gate 网络结构，用更经济的参数方式支撑更深层次的 Gate Net 与更大的 embedding 规模，从而兼顾效果与成本。
- 在工业级数据集上取得可观的 AUC 提升，并在正式上线后带来显著线上指标收益。

## 开源与竞赛

### Agent-R1 — 面向 Agent 系统的强化学习框架

参与开源项目 **Agent-R1**，作为核心贡献者之一，项目致力于为 agentic system 提供端到端强化学习训练框架：

- 扩展传统单轮强化学习训练，支持多轮对话、多工具调用等真实 Agent 场景。
- 修正主流强化学习算法（如 PPO、GRPO 等）的优势估计与损失计算，使之更适配大模型场景。
- 负责整体框架设计、关键的多轮 RAG 系统构建以及模型训练调优，目前项目已获得较多关注，GitHub star 数持续增长。

### Meta KDD Cup 2024 · CRAG: Comprehensive RAG Benchmark

参与 Meta KDD Cup 2024 CRAG 赛道，主要围绕复杂现实场景下的检索增强生成（RAG）系统构建：

- 针对比赛中真实网页信息冗余、噪声大、时效性强等挑战，设计 query 域分类与实体抽取流程，对不同域与实体采用差异化检索策略。
- 结合知识图谱，对检索结果进行结构化整合和结果转写，并利用大模型的自验证能力过滤不可靠信息，提升整体系统鲁棒性和可信度。
- 最终取得 **全球第四名**、复杂问题特别奖，并将方案整理为公开论文。
